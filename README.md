# 10th-web-study

## 🌲 브랜치 규칙
- **브랜치 명** : `week00_닉네임_missionN`
- **생성 방식**: 이전 미션 브랜치에서 파생하여 작업 내역을 누적합니다.
  - (ex: `week01_Elric_mission1` → `week01_Elric_mission2` → `week01_Elric_mission3`)

---

## 📌 미션 제출 방식 안내 (중요!)
이번 기수부터는 폴더를 여러 개 만들어 제출하는 것이 아니라, **브랜치를 이어가는 방식**으로 제출합니다. 아래 예시를 참고하여 진행해 주세요.

### 💡 예시: 3주차에 미션 3개를 수행할 경우
1. **미션 1 제출**
   - `main` 브랜치에서 `week03_닉네임_mission1` 브랜치를 새로 만들어 작업합니다.
   - 미션 1 완성 시 커밋 & 푸시 후 **PR을 올립니다.**
2. **미션 2 제출**
   - `week03_닉네임_mission1` 브랜치에서 파생된 `week03_닉네임_mission2` 브랜치를 만들어 작업합니다.
   - 미션 2 완성 시 커밋 & 푸시 후 **PR을 올립니다.** (미션 1 내용 포함)
3. **미션 3 제출**
   - `week03_닉네임_mission2` 브랜치에서 파생된 `week03_닉네임_mission3` 브랜치를 만들어 작업합니다.
   - 미션 3 완성 시 커밋 & 푸시 후 **PR을 올립니다.** (미션 1, 2, 3 내용 모두 포함)

> **Note:** 이 방식을 통해 총 3개의 PR에 각 미션까지의 누적 내용을 각기 다른 PR URL로 제출할 수 있게 됩니다.

## 📂폴더 구조
- 현재 진행하는 주차에 자기 닉네임으로 프로젝트를 생성합니다.
- 해당 프로젝트에서 실습을 진행해주시면 됩니다.
```
├── week01
│    └── Elric
│    └── README.md
├── week02
├── week03
├── week04
├── week05
├── week06
├── week07
├── week08
├── week09
├── week10
└── README.md
```

## ✅ Pull Request 규칙
- PR 제목: [week00] 닉네임 n주차 미션
- 내용 : 워크북 진행 및 실습 진행 간에 느낀 점 한마디

PR 작성 이후에 우측부분에서 Reviewer 및 Assignee 지정해주세요!

### Reviewer & Assignee
- Reviewer : 각 파트장만 지정
- Assignee : 본인(작성자) 지정

📍 **Merge 규칙 :
PR 병합은 스터디 진행 후, 해당 파트장이 최종 Merge (그 외 인원은 Merge 하지 않음)**

## ✅ 커밋 메시지 규칙
| 타입      | 설명 |
|-----------|------|
| feat      | 새로운 기능 추가 |
| fix       | 버그 수정 |
| docs      | 문서 수정 (README, 주석 등) |
| style     | 코드 스타일 변경 (포맷, 세미콜론 등) |
| refactor  | 리팩토링 (기능 변화 없음) |
| test      | 테스트 코드 추가 / 수정 |
| chore     | 빌드 설정, 패키지 관리 등 기타 작업 |
| build     | 빌드 관련 파일 수정 |
| revert    | 이전 커밋 되돌리기 |

`(ex: feat: 기능 추가)`
